Personal project

brain

A model-independent project brain for AI coding editors. It keeps durable intent, architecture, constraints, decisions, experiments, and current focus beside the source code.

What it solves

Session continuity

A new model or editor can reconstruct the project before editing.

Focused context

A small boot set loads first; deeper knowledge loads only when relevant.

Protected behavior

Every significant change begins with an impact check and ends with verification.

Structure

project-root/
├── PROJECT_BRAIN.md      # Universal entrypoint
├── AGENTS.md             # Optional thin editor adapter
└── project-brain/
    ├── MANIFEST.yaml     # Boot and on-demand loading
    ├── purpose/
    ├── glossary/
    ├── architecture/
    ├── features/
    ├── plans/
    ├── changes/
    ├── decisions/
    ├── state/
    ├── memory/
    └── rules/

Use it now

  1. Install this npm package globally once.
  2. Run project-brain init inside any project.
  3. Run project-brain validate to verify the generated structure.
  4. Replace unverified placeholders with facts from the target source code.
  5. Tell the AI editor to follow PROJECT_BRAIN.md.

CLI initialization and structural validation are ready. Automatic deep source analysis and complex-monorepo testing remain planned.

Core rules

  • Product intent comes from PURPOSE.md.
  • Inspected code represents current implementation.
  • Conflicts are reported, never silently resolved.
  • Plans preserve provenance but are not authority.
  • Secrets and production or customer data never enter project-brain.
